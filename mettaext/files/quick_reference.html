ğŸ”§ Pass 1: Semantic Structure

Input: Raw narrative (.txt)

Output: out_pass1_*.txt

What It Does

  • Detects dialogue with speakers
  • Extracts internal monologue (*thoughts*)
  • Splits narration by sentence
  • Identifies scene headers

Output Format

{type:dialogue, speaker:Vairis} "We should..."
{type:narration} The sun had set.
{type:internal_monologue, speaker:Senareth} "Why?"
{type:scene_header} # Part 2: Discovery
âœ“ When to use: First step for all narratives

Enhanced: pass2_enhanced.py

ğŸ”€ Pass 3: ZONJ Merging

Inputs: out_pass1_*.txt + out_pass2_*.metta

Output: zonj_*.json

Features

  • Future-proof attribute parsing
  • Tolerant to format drift
  • Preserves unknown header fields
  • Line-level inference attachment

Output Structure

{
  "type": "scene",
  "id": "first_contact",
  "segments": [
    {
      "line": 12,
      "type": "dialogue",
      "speaker": "Vairis",
      "text": "\"We should...\"",
      "inferred": {
        "emotion": [
          {"subject": "Vairis", "label": "caution", 
           "confidence": 0.9}
        ]
      }
    }
  ]
}

ğŸ’¾ Pass 4: ZON Bridge

Input: zonj_*.json

Outputs: *.zon + *.zonj.json

Required Metadata

--era FirstAge        # Temporal anchor
--location Beach      # Spatial anchor  
--scope narrative     # narrative/canon/lore/event
--output-dir ./zon    # Output directory

Optional Temporal Range

--start "T+0.0"       # Start time
--end "T+12.5"        # End time

Output: Human-Readable (.zon)

@id: scene.first_contact
@when: FirstAge.scene_001
@where: Realm/Physical/Beach
@scope: narrative
@entities: [senareth, vairis]

=segments:
  - line: 12
    type: dialogue
    speaker: Vairis
    text: "We should establish shelter,"

Output: Canonical JSON (.zonj.json)

{
  "@id": "scene.first_contact",
  "@when": "FirstAge.scene_001",
  "@where": "Realm/Physical/Beach",
  "@scope": "narrative",
  "@entities": ["senareth", "vairis"],
  "=segments": [...],
  "=inferred": {...}
}
